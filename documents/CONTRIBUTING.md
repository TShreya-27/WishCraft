# Contributing

Thanks for helping keep these documents useful and easy to maintain.

## Guidelines

- Keep changes focused and easy to review.
- Use plain language and consistent formatting.
- Prefer Markdown for editable documents.
- Add context when a document depends on a decision, date, source, or external reference.
- Remove outdated information instead of letting conflicting versions accumulate.

## Pull Requests

Before opening a pull request:

1. Check that filenames are clear and stable.
2. Make sure links work.
3. Confirm private or sensitive information is not included.
4. Summarize what changed and why.
