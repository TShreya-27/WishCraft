# Contributing

Keep changes focused and easy to review.

## Before You Commit

1. Run `npm run build`.
2. Check that no private Firebase credentials or local `.env` files are staged.
3. Keep generated folders such as `node_modules/` and `build/` out of git.
4. Add screenshots or notes when a UI change affects a user-facing screen.

## Style

- Follow the existing React component and CSS patterns.
- Use descriptive names for components, hooks, assets, and template files.
- Prefer small, direct changes over broad refactors.
